本指南将帮助你将 Logseq 知识库完整迁移到 Obsidian。

| 项目 | 路径 |

## 二、格式转换规则

### 2.1 大纲转标准 Markdown

Logseq 使用 `- ` 大纲格式，需要转换为标准 Markdown：

| Logseq 格式 | Obsidian 格式 |
|-------------|---------------|
| `- # 标题` | `## 标题` |
| `- ## 标题` | `### 标题` |
| `- ### 标题` | `#### 标题` |
| 普通列表项 | 保留 `- ` 或根据层级缩进 |
| 空行 `- ` | 删除 |

**示例：**
```
输入：- # 一、浏览器模块化存在的问题
输出：## 一、浏览器模块化存在的问题
```

### 2.2 内部链接

| 类型 | Logseq | Obsidian |
|------|--------|----------|
| 页面链接 | `[[页面名]]` | `[[页面名]]`（保持不变） |
| 日记链接 | `[[2024_10_11]]` | `[[2024-10-11]]` |

### 2.3 块引用

Logseq 的 `((uuid))` 块引用需要转换：

| Logseq | Obsidian |
|--------|----------|
| `((uuid))` | `[[当前页面名#^uuid]]` |

**示例：**
```
输入：((66c9da55-0766-452f-94ce-52e29d13112b))
输出：[[一 Vue框架设计#^66c9da55-0766-452f-94ce-52e29d13112b]]
```

### 2.4 标签

| Logseq | Obsidian |
|--------|----------|
| `#[[带空格的标签]]` | `#带空格的标签` |
| `#标签` | `#标签`（保持不变） |

### 2.5 任务状态

| Logseq | Obsidian |
|--------|----------|
| `- TODO 内容` | `- [ ] 内容` |
| `- DOING 内容` | `- [ ] 内容 #status/doing` |
| `- DONE 内容` | `- [x] 内容` |
| `- LATER 内容` | `- [ ] 内容 #status/later` |
| `- WAITING 内容` | `- [ ] 内容 #status/waiting` |

### 2.6 属性与高亮

| Logseq | Obsidian |
|--------|----------|
| `id:: uuid` | 删除或转为 YAML frontmatter |
| `[[$red]]==高亮文本==` | `==高亮文本==` |

### 2.7 资源路径

| Logseq | Obsidian |
|--------|----------|
| `../assets/xxx.png` | `assets/xxx.png` |

### 2.8 日记文件

| 项目 | Logseq | Obsidian |
|------|--------|----------|
| 目录 | `journals/` | `日记/` 或根目录 |
| 文件名 | `2024_10_11.md` | `2024-10-11.md` |

## 三、迁移步骤

### 步骤 1：创建目标目录结构

```bash
mkdir -p "/Users/guoyiding/Desktop/Obsidian仓库"
mkdir -p "/Users/guoyiding/Desktop/Obsidian仓库/assets"
mkdir -p "/Users/guoyiding/Desktop/Obsidian仓库/日记"
```

### 步骤 2：复制资源文件

```bash
cp -r "/Users/guoyiding/Desktop/第二大脑/assets/"* "/Users/guoyiding/Desktop/Obsidian仓库/assets/"
```

### 步骤 3：转换页面文件

将 `pages/` 目录下的 `.md` 文件按以下规则转换：

1. 大纲格式转标准 Markdown
2. 转换块引用
3. 转换任务状态
4. 删除 `id::` 属性
5. 转换标签格式

### 步骤 4：转换日记文件

将 `journals/` 目录下的文件：

1. 重命名：`2024_10_11.md` → `2024-10-11.md`
2. 转换内容格式（同页面文件）
3. 移动到 `日记/` 目录

### 步骤 5：验证链接

在 Obsidian 中打开目标仓库，检查：
- 双向链接 `[[页面名]]` 是否有效
- 图片资源是否正常显示
- 标签是否正确识别

### 步骤 6：清理

删除 Logseq 特有属性：
- `id::` 行
- `collapsed::` 属性
- 其他 Logseq 元数据

## 四、小范围试跑

建议先转换 1-2 个文件验证效果：

1. 选择一个简单的页面文件
2. 手动或使用脚本转换
3. 在 Obsidian 中打开检查
4. 确认无误后再批量执行

---

## 五、注意事项

1. **块引用限制**：Obsidian 不原生支持 `((uuid))` 块引用，转换后需要手动维护或使用插件
2. **白板**：Logseq 的白板（whiteboards）需要单独处理或放弃
3. **插件兼容**：部分 Logseq 插件功能在 Obsidian 中可能需要替代插件
4. **双向链接**：转换后建议在 Obsidian 中全面检查双向链接和标签

## 六、转换示例

### 示例 1：大纲与标题

**Logseq 输入：**
```
- # 一、浏览器模块化存在的问题
	- 效率问题：精细的模块划分...
	- 兼容性问题：...
```

**Obsidian 输出：**
```markdown
## 一、浏览器模块化存在的问题
  - 效率问题：精细的模块划分...
  - 兼容性问题：...
```

### 示例 2：块引用

**Logseq 输入：**
```
- vdom如何生成
  - ((66c9da55-0766-452f-94ce-52e29d13112b))
```

**Obsidian 输出：**
```markdown
- vdom如何生成
  - [[一 Vue框架设计#^66c9da55-0766-452f-94ce-52e29d13112b]]
```

### 示例 3：任务与属性

**Logseq 输入：**
```
- DONE [nextTick 实现原理](https://...)
- TODO 这里的 Watcher 是不是就是 track 函数
  id:: 66cc726b-e813-423f-bc72-43dfa689c1d9
```

**Obsidian 输出：**
```markdown
- [x] [nextTick 实现原理](https://...)
- [ ] 这里的 Watcher 是不是就是 track 函数
```
