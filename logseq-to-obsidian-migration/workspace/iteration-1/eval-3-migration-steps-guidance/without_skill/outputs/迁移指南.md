# Logseq 到 Obsidian 迁移指南

本指南将帮助您将 Logseq 知识库迁移到 Obsidian。请按照以下步骤操作。

---

## 一、迁移前准备

### 1.1 备份原始数据
在开始迁移之前，**务必**创建 Logseq 仓库的完整备份：
- 复制整个 Logseq 文件夹到其他位置
- 建议使用日期命名备份，如 `logseq-backup-2026-03-14`

### 1.2 安装 Obsidian
- 从 [Obsidian 官网](https://obsidian.md/) 下载并安装
- 首次启动时选择「创建新仓库」或直接打开一个空文件夹

---

## 二、格式转换

### 2.1 文件结构差异

| 特性 | Logseq | Obsidian |
|------|--------|----------|
| 日记位置 | `journals/` 文件夹 | 根目录或 `dailynotes/` |
| 页面位置 | `pages/` 文件夹 | 根目录 |
| 文件命名 | YYYY_MM_DD.md (日记) | YYYY-MM-DD.md (日记) |
| 资源文件 | `assets/` 文件夹 | `attachments/` 或自定义 |

### 2.2 内容格式转换

#### 2.2.1 待办事项转换
Logseq 格式：
```
- TODO 任务内容
- DOING 进行中的任务
- DONE 已完成的任务
```

Obsidian 格式：
```
- [ ] 任务内容
- [/] 进行中的任务（需要 Tasks 插件）
- [x] 已完成的任务
```

#### 2.2.2 块引用转换
Logseq 格式：
```
((block-id)) 内部块引用
```

Obsidian 格式：
```
[[文件名#^block-id]] 块引用
```

#### 2.2.3 标签转换
Logseq 格式：
```
#标签 或 [[标签]]
```

Obsidian 格式：
```
#标签 或 [[标签]]
```
> 注：两者兼容，无需转换

#### 2.2.4 属性转换
Logseq 格式：
```
- property:: value
```

Obsidian 格式（YAML frontmatter）：
```
---
property: value
---
```

### 2.3 自动化迁移脚本

可以使用以下工具进行自动迁移：
- **logseq-to-obsidian**: GitHub 上的开源迁移工具
- **自定义 Python 脚本**: 根据需求编写转换逻辑

## 三、迁移步骤

### 步骤 1：复制文件
```bash
# 将 Logseq 的 pages 文件夹内容复制到 Obsidian 仓库根目录
cp -r /path/to/logseq/pages/* /path/to/obsidian/vault/

# 复制日记文件（需要重命名）
cp -r /path/to/logseq/journals/* /path/to/obsidian/vault/
```

### 步骤 2：重命名日记文件
将日记文件从 `YYYY_MM_DD.md` 格式重命名为 `YYYY-MM-DD.md`：
```bash
# macOS/Linux
cd /path/to/obsidian/vault
for f in ????_??_??.md; do
  mv "$f" "$(echo $f | tr '_' '-')"
done
```

### 步骤 3：转换文件内容
1. 使用文本编辑器（如 VS Code）批量替换
2. 或使用迁移脚本自动处理

### 步骤 4：迁移资源文件
```bash
# 复制 assets 文件夹
cp -r /path/to/logseq/assets /path/to/obsidian/vault/attachments
```

### 步骤 5：更新资源链接
将 `../assets/` 路径替换为 `attachments/` 或您设置的路径。

## 四、Obsidian 配置

### 4.1 安装推荐插件
在 Obsidian 中安装以下插件：
- **日记 (Daily Notes)**: 管理日记
- **任务 (Tasks)**: 高级任务管理
- **Dataview**: 数据查询（类似 Logseq 查询）
- **模板 (Templates)**: 页面模板

### 4.2 配置日记格式
设置 → 日记 → 日期格式：`YYYY-MM-DD`

### 4.3 配置文件与链接
设置 → 文件与链接：
- 新建笔记的存放位置：仓库根目录
- 附件默认存放路径：`attachments`

## 五、迁移后检查

### 5.1 检查清单
- [ ] 所有页面文件已迁移
- [ ] 所有日记文件已迁移并重命名
- [ ] 待办事项格式正确显示
- [ ] 内部链接可以正常跳转
- [ ] 图片等资源文件正常显示
- [ ] 标签可以正常使用

### 5.2 常见问题解决

**问题 1：链接无法跳转**
- 检查文件名是否与链接中的名称一致
- 注意中英文标点符号差异

**问题 2：图片无法显示**
- 检查图片路径是否正确
- 确认图片文件已复制到正确位置

**问题 3：待办事项格式异常**
- 手动检查并修复遗漏的 TODO 格式
- 考虑使用 Tasks 插件增强功能

## 六、进阶建议

### 6.1 使用 Dataview 替代 Logseq 查询
Logseq 查询：
```
{{query (and [[标签]] (task LATER))}}
```

Obsidian Dataview：
```dataview
TASK
FROM #标签
WHERE !completed
```

### 6.2 建立新的工作流
- 设置快捷键提高效率
- 创建模板文件统一格式
- 使用看板视图管理任务
