# Obsidian 文件夹分析

分析 Obsidian 仓库中的文件夹，根据 Markdown 文件第一行的标签进行分类汇总。

## 分析流程

### 1. 获取文件列表

使用 `ls` 命令列出目标目录下的所有文件：

```bash
ls -la "/path/to/folder"
```

### 2. 提取标签

读取每个 `.md` 文件的第一行，提取 `#标签` 内容：

```bash
find "/path/to/folder" -maxdepth 1 -name "*.md" -exec sh -c 'echo "=== $1 ===" && head -1 "$1"' _ {} \;
```

**标签格式**：通常为 `- #标签名` 或 `#标签名`

**多标签**：一个文件可能有多个标签，如 `- #Vue #面经`

### 3. 分类汇总

按标签对文件进行分组，生成分类报告。

## 输出格式

### 按标签分类

```markdown
### 标签名

| 文件名 | 大小 |
|--------|------|
| file1.md | 2.3KB |
| file2.md | 5.1KB |
```

### 统计表格

```markdown
| 分类 | 文件数 | 占比 |
|------|--------|------|
| 标签1 | 10 | 25% |
| 标签2 | 8 | 20% |
```

## 特殊情况处理

1. **空文件**：第一行为空或无标签的文件，归入"无标签/空文件"类别
2. **多标签**：文件有多个标签时，在主要标签下列出，其他标签作为备注
3. **子目录**：默认只分析一级目录下的 `.md` 文件，不递归处理子目录

## 输出示例

